0エグゼクティブサマリー

アポドリは「営業の属人化」という普遍的な課題をAIエージェントで解決し、リリース1年で累計3,000件超の問い合わせを獲得した急成長サービス。しかし、その裏側は「Dify + 複数LLM + 分散処理」というシンプルな構成であり、技術的な参入障壁は高くない。真の参入障壁はDMMグループの資本力（20億円シード）とブランド力にある。

VAIZOが同領域で勝負するなら、正面からぶつかるのではなく、「AI導入支援で顧客の営業課題を掴む → 自社で営業AIエージェントをSaaS化 → M&A対象企業に組み込んで事業価値を上げて売却」というバリューチェーンの中に組み込む戦略が合理的。アポドリが「汎用ツール」として広く浅く展開する中、VAIZOは「特定業界の深い知見 × AI」で差別化できる余地がある。

市場機会
営業AIエージェント市場は急拡大中。アポドリの料金非公開・サービス歴の浅さは後発にとってチャンス。特に業界特化型は未開拓。
技術的実現性
アポドリの技術スタック（Dify + Claude/GPT/Gemini + Azure）は公開情報が豊富。CTO櫻井氏のLLM研究知見で再現・改良が可能。
VAIZOへの示唆
AI導入支援の現場で得た「顧客の営業課題」をSaaS化する流れは、VAIZOバリューチェーンのPhase 1→2そのもの。
最大のリスク
DMMの資本力によるマーケティング物量（タクシーCM等）には資本段階が異なるため正面勝負は不利。ニッチ戦略が必須。

「アポが取れるのはあの人だけ」状態。退職リスクが経営リスクに直結。営業ノウハウが組織に残らない。

アポドリの提唱する「営業3.0」: 営業活動の約80%を占める「情報収集」をAIが担い、人は「顧客に信頼されるコミュニケーション」に集中するモデル。

  1. 判断条件の言語化: エラーパターンを具体化しプロンプトに反映。完璧でなくてもLLMの判断に委ねる実用主義。

5開発の背景と作り方

約1年間の0→1開発（2024年1月〜2025年1月）

段階やったこと学び
1. 手作業で営業代行チームが実際に営業代行を手動実行。タスクを分解して言語化。手作業がドメイン知識の習得に直結。
2. AIへの段階的置換人手→AI置き換え開始。Dify導入でプロンプトを外部化。非エンジニアもプロンプト調整可能に。
3. プロダクション化LLM品質確認の仕組み化、UI整備。エンジニア介在なしの運用を実現。
CTO菊池氏の「あえて作らない」哲学:
  • UI開発を後回し — 初期はスプレッドシートで運用。「UIの枠内で考えてしまう」固定化を防止。
  • 自動化も後回し — 手動で業務解像度を上げてからシステム化。
  • 「できるか不明」を最優先で検証 — AIで実現可能かを先に確かめる。

8戦略的示唆 — アポドリから何を学ぶか

アポドリの「本当の強み」と「隙」

本当の強み
  • DMMの資本力: 20億円シード、タクシーCM全国展開。これは技術では追いつけない。
  • 先行者利益: 累計3,000件超の問い合わせで得た営業データ＝学習資産。
  • 「手作業→AI化」の開発手法: ドメイン知識を深く持った上での自動化は、精度が高い。
隙（攻められるポイント）
  • 汎用型ゆえの浅さ: 全業界対応＝特定業界への深い理解がない。業界特化型に弱い。
  • 料金非公開: 問い合わせのハードルが高い。透明な価格設定で差別化可能。
  • 実績がグループ内に偏重: Algoage、DMM.comは身内。第三者の成功事例が薄い。
  • フォーム送信のグレーゾーン: 1,000通に1社の「連絡不要」返信。コンプライアンス意識の高い企業は敬遠する可能性。
  • 技術的参入障壁は低い: Dify + LLM API + Azure。スタックは公開済みで再現可能。

VAIZOのバリューチェーンとの接続点

アポドリの事業モデルは、VAIZOバリューチェーンの「Phase 1（AI導入支援）→ Phase 2（AI SaaS化）」に直結する。

VAIZOがAI導入支援で顧客企業の営業課題を発見 → その課題を解決する営業AIツールを自社SaaS化 → M&A対象企業に組み込んで事業価値（評価倍率）を上げて売却。 これはアポドリの「丸パクリ」ではなく、VAIZOのバリューチェーン全体の中で営業AI機能を「レバー」として使うという上位戦略。

市場のホワイトスペース

領域アポドリのカバー状況VAIZOの機会
業界特化型営業AI 全業界対応（広く浅い） 特定業界（例：建設、医療、士業）に深く特化したAI営業エージェントは未開拓。AI導入支援の現場知見が直接活きる。
M&A対象企業への組み込み 対象外（SaaS単体販売のみ） 買収した企業に営業AIを導入→売上拡大→高倍率売却。アポドリにはない「出口戦略としてのAI組み込み」が可能。
中小企業向け透明価格 料金非公開（問い合わせ必須） 月額○万円と明示することで、問い合わせのハードルを下げ、中小企業の即決を促せる。
商談〜クロージング支援 アポ獲得まで（商談以降は未対応） 「参謀AI」のコンセプトで商談準備・提案資料生成・クロージング支援まで一気通貫で対応できれば大きな差別化。

9Next Action — VAIZOとして何をするか

以下は、このレポートの調査結果から導いたVAIZOへの具体的な推奨アクション。優先度順に整理。

優先度: 高
1. アポドリに問い合わせ、実際に体験する
料金が非公開である以上、推測で戦略を立てても仕方がない。まずは「AI導入支援事業の新規顧客開拓ツール」として資料請求・商談を実施し、以下を確認する：
  • 実際の価格帯（月額いくらか、従量課金の単価）
  • セットアップの工数と時間
  • どこまでカスタマイズできるか（業界特化はどこまで可能か）
  • 契約期間の縛り、解約条件
自社で使ってみることが、競合理解と「自社で作るべきか」の判断材料になる。
担当: CEO橋本
期限: 1週間以内
コスト: 無料（資料請求・商談）
優先度: 高
2. AI導入支援の現場で「営業課題」を意識的に収集する
アポドリが「手作業で営業代行→ドメイン知識を獲得→AI化」のプロセスで成功したように、VAIZOもAI導入支援の現場で顧客の営業課題を体系的に記録する。これがPhase 2（SaaS化）の種になる。
  • 導入支援先企業のうち、営業に課題を持つ企業を特定
  • 「どの業界の、どの営業プロセスが、なぜうまくいっていないか」を構造化して記録
  • 3社以上で共通する課題が見つかれば、それがSaaS化の候補
担当: AI導入支援チーム
期限: 継続的（月次レビュー）
コスト: 既存業務の延長
優先度: 中
3. CTO櫻井氏とアポドリの技術スタックを分析・再現可能性を評価する
アポドリの技術スタック（Dify + Claude/GPT/Gemini + Azure分散処理）は公開情報が豊富。CTO櫻井氏のLLM研究知見を活かし、以下を評価する：
  • Difyでのワークフロー構築を小規模で試作（1〜2週間で可能）
  • マルチLLMの用途別使い分けを自社環境で検証
  • 「自社で作る場合のコスト」vs「アポドリを利用する場合のコスト」の比較
  • 業界特化型にカスタマイズする場合、どの技術レイヤーに手を入れる必要があるか
担当: CTO櫻井
期限: 2週間以内
コスト: LLM API利用料（数千円程度）
優先度: 中
4.「Build vs. Buy vs. Partner」の判断フレームワークを整理する
営業AI機能をVAIZOのバリューチェーンに組み込む方法は3つ。シード段階（500万円）の資本制約を踏まえて判断する：
選択肢メリットリスクコスト感
Build（自社開発）完全なカスタマイズ、差別化、利益率が高い開発工数、シード段階では資金不足の可能性月10〜30万円（API費用）+ 開発工数
Buy（アポドリ利用）即日利用可能、開発不要差別化困難、従量課金でコスト増のリスク月20〜50万円（推定）
Partner（提携・OEM）自社ブランドで提供可能、開発コスト分担依存リスク、交渉コスト要交渉

推奨: まずBuy（アポドリを自社の営業に使う）で実データを蓄積しながら、CTO櫻井氏がBuild（業界特化型の自社版）の技術検証を並行で進める。Phase 2移行時にBuildに切り替える。

担当: CEO橋本 + CTO櫻井
期限: 1ヶ月以内に方針決定
優先度: 低（中長期）
5. M&A対象企業への「営業AI組み込みモデル」を設計する
VAIZOのバリューチェーンにおけるPhase 3（マイクロM&A）での活用シナリオ：
  • 小規模事業を買収 → 営業AIを導入して売上を拡大 → SaaS型に転換して評価倍率を上げる → 売却
  • このモデルが成立するかを、1社目のM&A候補で具体的にシミュレーションする
  • 「営業AIを入れることでどれだけ売上が伸びるか」の数値予測が、M&Aの投資判断（IRR計算）に直結する
アポドリの事例データ（アポ獲得率8倍、受注率3倍等）は、このシミュレーションの参考値として使える。
担当: CEO橋本
期限: M&A候補が出てきた段階
